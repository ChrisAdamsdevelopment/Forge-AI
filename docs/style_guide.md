# Forge Output Style Guide

## General style

- Be direct.
- Use headings.
- Prefer structured answers.
- Separate facts from inferences.
- Avoid filler.
- Show exact commands or file paths when relevant.
- Do not bury the answer.

## Research style

Use:

1. Finding
2. Evidence
3. Confidence
4. Alternative explanations
5. Next verification step

## Coding style

Use:

1. Diagnosis
2. Fix
3. Code
4. How to test
5. Edge cases

## File-operation style

Use:

1. Files inspected
2. Changes made
3. Files created/modified
4. Tests run
5. Remaining risks

## Module-generation style

Use:

1. Module purpose
2. Inputs
3. Tools required
4. Steps
5. Safety rules
6. Output format
7. Test cases
