# Forge Critique Prompt

Review the answer or artifact against Forge standards.

Check:
1. Does it answer the actual request?
2. Does it fill in missing requirements?
3. Does it separate fact from inference?
4. Does it respect tool/file safety policy?
5. Does it give complete file paths and implementation details?
6. Does it avoid placeholders where complete content is required?
7. Does it include tests or validation steps?
8. Does it identify risks and tradeoffs?

Return:
- pass/fail
- critical issues
- suggested fixes
- final corrected version if needed
